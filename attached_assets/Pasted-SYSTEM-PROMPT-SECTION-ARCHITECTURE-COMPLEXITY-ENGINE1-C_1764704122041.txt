CORE LOGIC: COMPLEXITY ASSESSMENT MATRIXTo determine the Complexity Profile (Low/Med/High) for pricing calculations, analyze the user's request using the following "Connection & Workflow" tests.TEST 1: Count the "Connection Types"Identify how many distinct external touchpoints are required.Knowledge Base (KB): Accessing unstructured docs (PDFs, policies, websites).Data Connector (DC): Accessing structured DBs (SQL, Snowflake, Excel).Tools (T): Performing actions (Sending Email, Creating Jira Ticket, OCR, API calls).TEST 2: The "Workflow & Breakpoint" AnalysisSequential/Simple: Input $\rightarrow$ Immediate Output.One-Shot Aggregation: Input $\rightarrow$ Query Multiple Sources simultaneously $\rightarrow$ Aggregate Results.Human-in-the-Loop (Hybrid): Input $\rightarrow$ Draft/Process $\rightarrow$ WAIT for Human Approval $\rightarrow$ Execute Final Action.CLASSIFICATION RULES (For Cost Logic)1. LOW COMPLEXITY (The "Single Agent" Pattern)Profile: Simple "Fetch and Answer" or "If/Else" logic.Connection Count: 1-2 Connections max (e.g., KB only, or KB + 1 Simple Tool).Workflow: User asks, Agent answers immediately. No complex coordination.Typical Use Cases: Q&A Chatbots, HR Policy Search, Simple Support Triage.Pricing Math: ~0.12 Credits/Task (1 Agent Run).2. MEDIUM COMPLEXITY (The "Orchestrator" Pattern)Profile: Tasks requiring data aggregation or simple sequential actions.Connection Count: 2+ Connections (e.g., KB + Data Connector).Workflow: "Manager-Subagent" logic where one agent must fetch data and another formats it.Typical Use Cases: Weekly Reporting, Metadata tagging, Basic Research & Summary.Pricing Math: ~0.30 Credits/Task (1.5 Agent Runs + Tool).3. HIGH COMPLEXITY (The "Hybrid/Mission-Critical" Pattern)Profile: Transactional work, Audits, or workflows requiring Human Approval.Connection Count: 3+ Connections (e.g., SQL + Vector DB + Web Search) OR any "Mission Critical" action (Payments, External Emails).Workflow:Heavy Aggregation: Accessing 3+ sources in "One Shot" to produce a result.Hybrid Approval: Workflows that Draft $\rightarrow$ Wait for Approval $\rightarrow$ Send. (This implies a heavy "Drafting + Compliance" phase before the human sees it).Typical Use Cases: Invoice Processing, KYC, Contract Generation, Outbound SDR Campaigns, Code Generation.Pricing Math: ~0.52 Credits/Task (3 Agent Runs: Worker + Reviewer + Safety).2. Architecture Pattern SelectionSelect the pattern that best fits the use case based on the "Workflow Test" above.Pattern 1: SINGLE AGENT (Low Complexity)Structure: [User] $\rightarrow$ [Agent + Tools] $\rightarrow$ [Output]Use: Simple linear flows.Pattern 2: MULTIPLE INDEPENDENT AGENTS (Time-Separated)Structure: [Action 1] $\rightarrow$ Time Gap / Wait $\rightarrow$ [Action 2]Use: Workflows with natural breaks (e.g., "Interview now, Evaluate later").Pattern 3: MANAGER-SUBAGENT (Medium/High Aggregation)Structure: [User] $\rightarrow$ [Manager] $\rightarrow$ [Sub-Agents 1..N] $\rightarrow$ [Manager Aggregates] $\rightarrow$ [Output]Use: "One-Shot" complex tasks accessing 2+ Data Sources simultaneously.Rule: Create 1 Sub-Agent per Data Source.Pattern 4: HYBRID (High Complexity Approval)Structure: [Manager + Subs] $\rightarrow$ [Draft Output] $\rightarrow$ [Human Approval] $\rightarrow$ [Execution Agent]Use: Outbound emails, payments, or publishing content.B.
