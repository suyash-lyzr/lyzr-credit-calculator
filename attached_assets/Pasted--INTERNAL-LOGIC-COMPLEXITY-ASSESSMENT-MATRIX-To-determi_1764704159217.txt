### **INTERNAL LOGIC: COMPLEXITY ASSESSMENT MATRIX**
To determine the **Complexity Profile** (Low, Medium, or High), analyze the user's request using the following "Connection & Workflow" tests.

**TEST 1: Count the "Connection Types"**
Identify how many distinct external touchpoints are required.
* **Knowledge Base (KB):** Accessing unstructured docs (PDFs, policies).
* **Data Connector (DC):** Accessing structured DBs (SQL, Snowflake).
* **Tools (T):** Performing actions (Sending Email, Creating Jira Ticket, OCR).

**TEST 2: The "Workflow & Breakpoint" Analysis**
* **Sequential/Simple:** Input $\rightarrow$ Immediate Output.
* **One-Shot Aggregation:** Input $\rightarrow$ Query Multiple Sources simultaneously $\rightarrow$ Aggregate Results.
* **Human-in-the-Loop (Hybrid):** Input $\rightarrow$ Draft/Process $\rightarrow$ *WAIT for Human Approval* $\rightarrow$ Execute Final Action.

---

### **CLASSIFICATION RULES**

**1. LOW COMPLEXITY (The "Single Agent" Pattern)**
* **Profile:** Simple "Fetch and Answer" or "If/Else" logic.
* **Connection Count:** 1-2 Connections max (e.g., KB only, or KB + 1 Simple Tool).
* **Workflow:** User asks, Agent answers immediately. No complex coordination.
* *Typical Use Cases:* Q&A Chatbots, HR Policy Search, Simple Support Triage.
* *Pricing Math:* **~0.12 Credits/Task** (1 Agent Run).

**2. MEDIUM COMPLEXITY (The "Orchestrator" Pattern)**
* **Profile:** Tasks requiring data aggregation or simple sequential actions.
* **Connection Count:** 2+ Connections (e.g., KB + Data Connector).
* **Workflow:** "Manager-Subagent" logic where one agent must fetch data and another formats it.
* *Typical Use Cases:* Weekly Reporting, Metadata tagging, Basic Research & Summary.
* *Pricing Math:* **~0.30 Credits/Task** (1.5 Agent Runs + Tool).

**3. HIGH COMPLEXITY (The "Hybrid/Mission-Critical" Pattern)**
* **Profile:** Transactional work, Audits, or workflows requiring **Human Approval**.
* **Connection Count:** 3+ Connections (e.g., SQL + Vector DB + Web Search) OR any "Mission Critical" action (Payments, External Emails).
* **Workflow:**
    * **Heavy Aggregation:** Accessing 3+ sources in "One Shot" to produce a result.
    * **Hybrid Approval:** Workflows that Draft $\rightarrow$ Wait for Approval $\rightarrow$ Send. (This implies a heavy "Drafting + Compliance" phase before the human sees it).
* *Typical Use Cases:* Invoice Processing, KYC, Contract Generation, Outbound SDR Campaigns, Code Generation.
* *Pricing Math:* **~0.52 Credits/Task** (3 Agent Runs: Worker + Reviewer + Safety).
